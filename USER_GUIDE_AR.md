# دليل البدء السريع لمشروع "بوصلة العافية"

لقد قمت برفع كافة التعديلات على مستودع GitHub الخاص بك. إليك شرح مفصل لكيفية استخدام الملفات التي تم تسليمها والخطوات المطلوبة لتشغيل النظام بالكامل:

## 1. إعداد قاعدة البيانات (Supabase)
الملف الموجود في `supabase/migrations/20260220000000_full_schema.sql` هو حجر الزاوية لنظام البيانات.
- **ماذا تفعل؟** اذهب إلى لوحة تحكم Supabase الخاصة بمشروعك، ثم انتقل إلى قسم **SQL Editor**.
- **التنفيذ:** انسخ محتوى الملف وألصقه هناك ثم اضغط **Run**. سيقوم هذا بإنشاء كافة الجداول، الأنواع (Enums)، وسياسات الأمان (RLS) المطلوبة.

## 2. نظام المصادقة (Auth)
الملف `src/lib/auth.ts` يوفر دوال جاهزة للتعامل مع المستخدمين.
- **الاستخدام:** تم دمج هذا الملف بالفعل في صفحة التسجيل (`Register.tsx`) ليتم إنشاء المستخدمين حقيقياً في Supabase بدلاً من البيانات الوهمية.
- **ملاحظة:** تأكد من إعداد متغيرات البيئة (`.env`) الخاصة بـ `VITE_SUPABASE_URL` و `VITE_SUPABASE_PUBLISHABLE_KEY` في مشروعك.

## 3. التحول إلى تطبيق جوال (Capacitor)
الملف `capacitor.config.ts` هو الملف التعريفي لتحويل موقع الويب إلى تطبيق Android أو iOS.
- **الخطوة القادمة:** يمكنك الآن تشغيل الأوامر التالية في جهازك المحلي لبناء تطبيق الجوال:
    ```bash
    npm run build
    npx cap add android
    npx cap sync
    npx cap open android
    ```

## 4. متابعة التقدم (Tech Lead Progress)
الملف `TECH_LEAD_PROGRESS.md` هو تقريري لك كقائد تقني، يوضح ما تم إنجازه وما يجب التركيز عليه في المرحلة القادمة. يمكنك الرجوع إليه دائماً لمعرفة حالة المشروع الحالية.

## 5. دليل المستخدم (User Guide)
هذا الملف (`USER_GUIDE_AR.md`) هو مرجعك السريع للخطوات التقنية.
